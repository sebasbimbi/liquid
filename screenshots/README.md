# Screenshots

Drop preview screenshots here. README.md references `screenshots/dark.png` as the hero image.

## Recommended captures

Capture each at **2x retina** (`Cmd+Shift+4` then `Space` then click the window on macOS — captures the active window with shadow).

| File | Surface | What to show |
|---|---|---|
| `dark.png` | Hero | Dark mode, file explorer + active note open + tabs visible. Translucent window ON over a forest/shoreline wallpaper. |
| `light.png` | Hero alt | Light mode equivalent — different wallpaper (sunlit). |
| `tabs.png` | Tabs | Multiple tabs, one active with accent under-stripe. Shows hover state if possible. |
| `modal.png` | Strong glass | Quick switcher (`Cmd+P`) or settings modal — full vibrancy + shadow stack. |
| `code.png` | Syntax | A note with a code block showing the syntax palette. Use a multi-language example: TypeScript, CSS, or Python. |
| `graph-filter.png` | Glass card | Graph view's filter panel — shows the floating glass card pattern. |
| `file-tree.png` | File explorer | Deep folder nesting with chevrons + active file pill. |
| `callout.png` | Callout glass | A `> [!note]` or `> [!tip]` callout in reading view. |

## Tips

- Use a wallpaper with some color variance so the glass effect reads (solid black/white wallpapers make blur invisible).
- Keep traffic lights visible (top-left) so viewers know it's macOS.
- Use real content from your vault if shareable, otherwise use Obsidian's "Help" vault for clean placeholder content.
- Compress with [TinyPNG](https://tinypng.com) before pushing — keeps the repo light.
